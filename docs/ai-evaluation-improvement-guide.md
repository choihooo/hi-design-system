# AI 평가 시스템 개선 가이드

## 🎯 개선 개요

AI 친화성 평가 시스템을 100점 체계로 개선하여 더 명확하고 실용적인 평가를 제공합니다.

## 📊 새로운 평가 체계

### 기존 vs 개선된 체계 비교

| 구분 | 기존 체계 | 개선된 체계 |
|------|-----------|-------------|
| 최대 점수 | 22점 | 100점 |
| 등급 | 4단계 (A-D) | 6단계 (S-F) |
| 평가 항목 | 5개 | 3개 |
| 개선 제안 | 일반적 | 구체적 + 우선순위 |

### 새로운 평가 항목

1. **코드 품질 (50점)**
   - 명명 규칙 (25점)
   - 타입 안전성 (25점)
   - 구조적 설계 (하위 합계)

2. **유지보수성 (30점)**
   - 문서화 품질 (20점)
   - 구조적 유지보수성 (10점)

3. **일관성 (20점)**
   - 이벤트 핸들러 패턴 (10점)
   - 상태 관리 일관성 (5점)
   - Props 일관성 (5점)

## 🏆 등급 체계

| 점수 | 등급 | 설명 |
|------|------|------|
| 90-100 | 🏆 S | 최우수 |
| 80-89 | 🟢 A | 우수 |
| 70-79 | 🟡 B+ | 양호 |
| 60-69 | 🟠 B | 보통 |
| 40-59 | 🔴 C | 개선필요 |
| 0-39 | ⚫ D | 불량 |

## 📈 사용 방법

### 1. 단일 컴포넌트 평가

```bash
# 기본 평가
pnpm ai-score packages/react/src/components/Button/Button.tsx

# 결과 예시
📁 파일: packages/react/src/components/Button/Button.tsx
🏆 최종 점수: 85/100 (🟢 A (우수))

📈 새로운 평가 구조:
코드 품질: 45/50
유지보수성: 25/30
일관성: 15/20

💡 개선 제안:
1. [Consistency] Event Handler Unification
   우선순위: 🟡 중간
   • onClick → handleClick (explicit)
   • Same pattern across all components
```

### 2. 전체 패키지 평가

```bash
# 전체 패키지 평가
```

## 🔧 개선된 기능

### 1. 자동화된 개선 제안

- **우선순위 부여**: High, Medium, Low
- **카테고리 분류**: 코드 품질, 유지보수성, 일관성
- **구체적인 예시**: 실제 코드 변경 사례 제공

### 2. 호환성 유지

- 기존 평가 구조와 신규 평가 구조 동시 표시
- 기존 스크립트 계속 사용 가능
- 점수 변환 알고리즘 내장

### 3. 시각적 개선

- 진행바 (████░░░) 표시
- 이모지 기반 등급 시스템
- 명확한 결과 분류
